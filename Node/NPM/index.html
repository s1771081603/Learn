认识 NPM

  1. npm（全称 Node Package Manager，即 node 包管理器）
  2. 是 Node.js 默认的，以 JavaScript 编写的软件包管理系统
  3. npm 来分享和使用代码已经成了前端的标配
  4. 官网：https://www.npmjs.com
  5. npm 被全球超过一百万开发人员所依赖
  6. 拥有超过一百万个软件包，是世界上最大的软件注册表

NPM 的使用

  1. npm -v 通过查看版本号，判断 npm 是否安装成功
  2. npm install <Module Name> 使用 npm 命令安装模块，可以直接在命令行里使用
  3. npm list -g 查看所有全局安装的模块
  4. npm list vue 查看某个模块的版本号
  5. npm -g install npm@5.9.1（@后面跟版本号）这样我们就可以更新 npm 版本
  6. npm install --save moduleName # --save 在 package 文件的 dependencies 节点写入依赖
  7. npm install --save-dev moduleName # --save-dev 在 package 文件的 devDependencies 节点写入依赖
  8. dependencies：运行时的依赖。发布后，即生产环境下还需要用的模块
  9. devDependencies：开发时的依赖。里面的模块是开发时用的，发布时用不到他，比如项目中使用的 gulp ，压缩css，js 的模块。这些模块在我们的项目部署后是不需要的

Package.json 属性说明

  1. name - 包名
  2. version - 包的版本号
  3. description - 包的描述
  4. homepage - 包的官网 url
  5. author - 包的作者姓名
  6. contributors - 包的其他贡献者姓名
  7. dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
  8. repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可以在 github 上。
  9. main - main 字段指定了程序的入口文件，require（`moduleName`）就会加载这个文件。这个字段的默认值是模块根目录下的 index.js。
  10. keywords - 关键字
  11. package.json 文件中版本号的说明，安装的时候代表不同的含义：
    1. "5.0.3" 表示安装指定 5.0.3 版本
    2. "~5.0.3" 表示安装 5.0.X 中最新版本
    3. "^5.0.3" 表示安装 5.X.X 中最新版本

包的使用

  1. CommonJS 中很大的一部分便是对模块系统化的规范
  2. 使用 require 语句导入包
  3. 新的 ES6 可以使用 import 导入包
